DungTrack 2 documentation

DungTrack 2 provides autotracking and basic track analysis for dung beetle orientation experiments. This software is designed for arena-based experiments. It will work for other types of experiment but the feature set, user interface, and available analysis options are designed for arena-based experiments. This software is supported for Linux (Ubuntu) and MacOS. Windows is not 'offically' supported but if you know how to use Python, Pip, and virtual environments on Windows then you can almost certainly get it running. Alternatively, you could try using the Windows Subsystem for Linux.

Why HTML?

This documentation is written in basic HTML/CSS so that

  1. Everyone using the software will have access to software which can render the documentation (a web browser).
  2. Updating the source is relatively straightfoward.
  3. Instructions can be divided into different linked pages rather than a single monolithic README.

Rough style key

Regular text appears like this. All of the core stuff you need to understand should be written in this plain fashion. Terminal commands appear like this:

$ command arg1 arg2

The $ sign is included by convention to indicate a terminal prompt, you should copy everything after the $ sign if you want to run the commadn (don't run them blindly). On some systems this symbol can be different (e.g. %).
