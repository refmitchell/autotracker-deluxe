Parameter file and project files

Back to main page

The software stores information about its own configuration and information about each project in JSON files.

You can edit these files manually but shouldn't.

These files can provide a local state on your machine which might mean something that worked during development doesn't work for you. If this happens, you should file a bug report.

Parameter file

An example parameter file looks like this: 


  {
  "project_directory": "/home/robert/autotracker-deluxe/data/p01",
  "project_file": "/home/robert/autotracker-deluxe/data/p01/p01.dt2p",
  "options.autotracker.track_point": "centre-of-mass",
  "options.autotracker.cv_backend": "BOOSTING",
  "options.autotracker.bg_computation_method": "random_N_mean",
  "options.autotracker.bg_sample_size": 20,
  "options.video.directory": "/home/robert/postdoc/source",
  "options.autocalibration.fix_k1": true,
  "options.autocalibration.fix_k2": true,
  "options.autocalibration.fix_k3": true,
  "options.autocalibration.fix_tangential": true,
  "options.autocalibration.show_meta_text": false,
  "options.processing.plot_grid": true,
  "options.processing.include_legend": true,
  "options.processing.filename": "processed_tracks",
  "options.processing.filetype": "pdf",
  "options.processing.zero": true,
  "options.autotracker.track_interval": 1,
  "options.autotracker.remember_roi": true,
  "options.autotracker.show_roi": false,
  "options.processing.smoothing_spline_degree": 3,
  "options.processing.smoothing_scale_factor": 0.3
  }
Occassionally, things can go wrong with the params file. This is usually because an item has been removed, or a new item has been added incorrectly during development. In these cases it's usually easiest to delete the file and let the software build a new one.
